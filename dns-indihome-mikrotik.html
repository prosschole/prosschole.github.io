Mikrotik

adalah sistem operasi dan perangkat lunak yang dapat digunakan untuk menjadikan komputer manjadi router network yang handal, mencakup berbagai fitur yang dibuat untuk IP network dan jaringan wireless, cocok digunakan oleh ISP, provider hotspot dan warnet.

Fungsinya

Pengaturan koneksi internet dapat dilakukan secara terpusat dan memudahkan untuk pengelolaannya.

Konfigurasi LAN dapat dilakukan dengan hanya mengandalkan PC Mikrotik Router OS dengan hardware requirements yang sangat rendah.

Blocking situs-situs terlarang dengan menggunakan proxy di mikrotik.

Pembuatan PPPoE Server.

Billing Hotspot.

Memisahkan bandwith traffic internasional dan local, dan lainnya.

Jenisnya

- Mikrotik RouterOS

- Mikrotik RouterBoard

Cara Menginstalnya

- download mikrotik

- siapkan Mikrotik RouterBoard

Burn file ISO MikroTiknya ke CD.

Masukkan cd mikrotik ke dalam cd/dvd room.

- Setting bios komputer anda, pada booting awal (first boot)nya adalah cd/dvd room anda.

- Setelah di setting maka komputer anda akan booting pertama kale ke cd/dvd room anda.

- Lakukan proses instalasi Mikrotik dengan memilih (check) semua pilihan yang ada dengan tombol ‘a‘. Kalo udah dicentang semua tekan tombol ‘i’ untuk meginstall Mikrotik.

- Tunggu bentar sampe proses instalasi nya selesai. Sabar yahh, cuma bentar kok. Kalo udah selesai tekan tombol Enter untuk reboot.

- Setelah komputer anda restart maka anda akan diminta untuk check disk atau tidak. Klik tombol “Y” untuk menyetujui, atau klik tombol “N” untuk tidak menyetujui.

Setelah itu akan muncul tampilan login. “admin” (tanpa tanda petik) pada Mikrotik - - - Login. Dan pada password enter aja. karena password defaultnya tidak ada password.

- Jika berhasil maka akan muncul gambar tampilan awal mikrotik (lihat gambar dibawah ini). Menandakan anda telah berhasil menginstal mikrotik.

11. apa yang dimaksud dengan dns dan apa perbedaan public dns dengan dns default


DNS yaitu Sebuah sistem yang menyimpan informasi tentang nama host ataupun nama domain dalam bentuk basis data tersebar (distributed database) di dalam jaringan komputer.

Jawaban:

DNS atau Domain Name System berfungsi sebagai penerjemah/pengubah dari alamat situs ke bentuk IP address.

Simplenya DNS adalah buku telepon agar perangkat kita dapat menemukan IP address dari suatu situs web dengan lebih cepat.

Contoh DNS

Beberapa DNS publik yang sudah sangat umum yaitu DNS Google (8.8.8.8) dan OpenDNS (208.67.222.222). Banyak penyedia Public DNS lainnya yang memberikan layanan DNS secara gratis seperti Cloudflare, AdGuard, Quad9, Verisign, dll. yang menyediakan keunggulannya masing-masing.

Semua tergantung User dalam memilih ingin menggunakan DNS yang mana.

Perbedaan

DNS Publik atau Public DNS memiliki beberapa keunggulan tersendiri tergantung dari penyedia DNS-nya. Terkadang menggunakan Public DNS jauh lebih menguntungkan dalam beberapa hal dibandingkan menggunakan DNS default bawaan provider.

Contoh Penggunaan

Menggunakan Public DNS seperti Google DNS lebih menguntungkan daripada menggunakan DNS bawaan provider karena server Google sendiri sudah sangat canggih, jauh lebih maju dibanding server yang digunakan provider internet tertentu sehingga dalam beberapa kasus jika kita menggunakan Google DNS atau DNS publik lainnya maka aktivitas browsing kita bisa jauh lebih lancar dan cepat.

Lalu biasanya DNS bawaan dari provider internet kita sudah dimodifikasi agar menampilkan iklan yang mempromosikan produk & layanan mereka. Beberapa pengguna pastinya merasa terganggu dengan iklan-iklan ini. Hal ini tentunya tidak ada jika menggunakan Public DNS dari Google, OpenDNS, Cloudflare, dsb.

Catatan

Yang perlu diperhatikan adalah pastikan untuk selalu menggunakan Public DNS dari penyedia yang memang terpercaya seperti Google, Cloudflare, OpenDNS, dan sejenisnya. Hal ini karena aktivitas browsing dapat dicatat oleh server DNS. Agar tidak disalah-gunakan maka harus menggunakan penyedia Public DNS yang terpercaya.

Fungsi Mikrotik:

Routing: Mikrotik dapat digunakan untuk routing jaringan, yaitu menentukan jalur yang akan digunakan oleh paket data untuk mencapai tujuan yang ditentukan.Firewall: Mikrotik dapat digunakan untuk mengamankan jaringan dengan mengatur pengaturan firewall yang akan mengontrol akses jaringan.VPN: Mikrotik dapat digunakan untuk menyediakan koneksi VPN (Virtual Private Network) yang aman untuk menghubungkan jaringan yang berbeda.Wireless: Mikrotik dapat digunakan untuk mengatur jaringan wireless, seperti membuat hotspot atau mengatur pengaturan wireless.Quality of Service (QoS): Mikrotik dapat digunakan untuk mengatur prioritas paket data untuk menjamin kualitas jaringan.

Bagian Mikrotik:

CPU: Bagian yang bertanggung jawab untuk menjalankan perintah-perintah dari sistem.RAM: Bagian yang digunakan untuk menyimpan data sementara saat proses yang sedang berjalan.Flash memory: Bagian yang digunakan untuk menyimpan data permanen seperti firmware Mikrotik.Ethernet ports: Bagian yang digunakan untuk menghubungkan perangkat jaringan ke Mikrotik.USB ports: Bagian yang digunakan untuk menghubungkan perangkat tambahan seperti modem atau hard drive eksternal.Power jack: Bagian yang digunakan untuk menghubungkan Mikrotik dengan sumber listrik.

21. Kenapa beberapa website tidak bisa dibuka dengan wifi indihome?


Jawaban:

Beberapa penyebab website tidak dapat diakses, diantaranya :

Setting DNS dan Name Server

Jika website anda tidak bisa dibuka pada semua komputer, termasuk laptop, maka kemungkinan ada kesalahan ketika anda melakukan pengaturan DNS dan Name Server. Kesalahan ini bisa menyebabkan website anda tidak bisa diakses beberapa jam hingga beberapa hari. Solusi yang bisa anda lakukan untuk mengatasi kesalahan setting DNS dan NS adalah dengan memperbaikinya dengan alamat NS yang benar. Untuk melakukan perbaikan ini anda harus memiliki akses ke halaman domain panel, bukan CPanel Hosting.

File Index yang tidak ada.

Ketika anda mencoba untuk mengakses website anda, maka server akan memanggil beberapa file berikut ini pertama kali, seperti: index.htlm, index.htm atau index.php (biasanya server akan mencari file dengan urutan seperti itu). Jika server tidak menemukan file tersebut, maka server akan menampilkan pesan error kepada anda (pengunjung). Beberapa pesan error yang akan ditampilkan adlaah error document, index file atau bahkan server akan menampilkan semua yang terdapat di direktori tersebut.

Jika pada website anda sudah terdapat salah satu dari file diatas, namun masih error, maka mungkin saja file tersebut dalam keadaan kosong. Untuk melihatnya anda harus membukanya melalui CPanel Hosting anda. Jika file index dalam keadaan kosong, biasanya halaman yang akan muncul adalah halaman berwarna putih tanpa tampilan yang lain. Solusinya bisa dengan cara Login ke CPanel Hosting anda, apakah di direktori website anda terdapat file tersebut atau tidak.

Pastikan juga apakah ada isinya berupa code atau tidak. Jika tidak, maka anda harus memperbaikinya file tersebut dan mengetikan code tertentu, jika anda menggunakan wordpress, anda tinggal upload ulang file index dari file wordpress anda.

Nama Domain yang sudah kadaluarsa (Expire date)

Sebenarnya tidak ada waktu kadaluarsa dari domain, jika anda membayarnya tepat waktu. Jadi, hal ini terjadi karena anda terlambat memperpanjang domain tersebut. Oleh karenya jangan pernah terlambat memperpanjang domain anda. Jika anda tidak memperpanjang domain tepat waktu, maka domain tersebut akan dikunci dan diarahkan sebuah halaman (halaman parking). Pada masa ini sebenarnya anda masih bisa memperpanjang domain dengan harga normal. Namun setelah masuk ke masa delete, maka harga perpanjangan domain akan sangat mahal. Solusi dengan cara memperpanjang domain anda tersebut hingga beberapa waktu ke depan (minimal 1 tahun)

IP anda terblokir Server

Website anda tidak bisa diakes di internet bisa juga karena IP Komputer anda terblokir oleh server. Blokir ini bisa terjadi karena beberapa hal berikut :

1) Salah Login CPanel/Webmail,FTP/SSH hingga berulang-ulang

2) Terlalu sering me-refresh browser

3) Terdeteksi melakukan tindakan spam

4) Hacking

5) DOS Attack

6) Port Scan

Komputer anda terinfeksi Virus

Internet banyak mendatangkan manfaat, namun pada saat yang sama juga masalah. Saat ini banyak sekali program jahat yang mencoba untuk menghalangi komputer anda untuk mengakses internet.

Kesalahan mengedit File .htaccess

Kesalahan dalam menulis kode di .htacces juga bisa menyebabkan website anda tidak bisa diakses. Oleh karenanya, ketika anda ingin merubah kode-kode yang terdapat di htaccess, pastikan anda melakukannya dengan benar atau lakukan backup file tersebut terlebih dahulu.

27. ketika kita selesai setting mikrotik 1 lalu kita lanjut mikrotik ke 2 setelah itu kenapa setingan mikrotik 1 masih ada pada setingan mikrotik ke 2​


Penjelasan:

Jika kmu melakukan konfigurasi pada dua perangkat MikroTik yang terpisah, pengaturan pada satu perangkat tidak akan secara otomatis muncul pada perangkat MikroTik yang lain. Meskipun pengaturan di kedua perangkat dapat diatur agar sama atau mirip, namun secara default tidak ada hubungan antara dua perangkat MikroTik yang terpisah.

Mungkin ada beberapa alasan mengapa kmu merasa bahwa pengaturan pada MikroTik pertama muncul pada MikroTik kedua. Salah satu kemungkinan adalah bahwa kmu telah melakukan kloning atau mengimpor konfigurasi MikroTik pertama ke MikroTik kedua, yang mengakibatkan pengaturan pada kedua perangkat menjadi identik. Namun, tanpa tindakan seperti itu, kedua perangkat MikroTik akan memiliki pengaturan yang terpisah dan independen satu sama lain.

29. bagaimana cara kerja dns dan dns server


DNS bekerja dengan menggunakan protokol UDP.Tujuanya adalah untuk performance.DNS pada saat client melakukan request terhadap DNS tujuan, yang pertama dilakukan mengecek pada komputer apakah request sudah ada di cache atau belum, jika ada buka, jika tidak kirim ke server, yang dikirimkan oleh Client, menggunakan query yang dikirim, setelah itu akan ada balasan dan diakses oleh browser.

Pada saat DNS Server menerima request, akan mencari domain (resolve) terhadap cache nya, jika sudah ada maka kembalikan ke client, jika tidak ada dicari ke server lain, namun jika pada detik tertentu, akan kemungkinan browser menyatakan timeout karena server terlalu lama merespon.
