10. Sebutkan jenis/macam perintah SQL (CRUD) serta berikan contoh untuk masing masing jenis perintah tersebut 2 SQL.​


Jawaban:

Structured Query Language atau SQL mungkin merupakan poin penting dalam sebuah studi teknik komputer jaringan. Keberadaan SQL sebagai sebuah bahasa pemrograman memiliki sejarah dan perkembangan yang panjang. Tentu saja perkembangan tersebut dipengaruhi oleh perkembangan dunia teknologi yang dari tahun ke tahun semakin membaik dan terus diperbaharui. Sehingga mau tidak mau, banyak software diciptakan dengan teknologi baru namun tidak menghilangkan keberadaan SQL ini.

Lalu, mungkin sebagian dari kamu mungkin bertanya-tanya apa pengertian SQL itu sendiri? Apa saja fungsi dengan menggunakan SQL ini dan apa saja perintah dasarnya yang perlu kamu ketahui? Semuanya bisa kamu simak ulasannya dalam tulisan di poin-poin berikut ini.

Sekilas mengenai SQL

Pengertian SQL secara umum bagi kalangan awam maupun pemula dalam dunia IT bisa diartikan sebagai bahasa yang digunakan dalam mengakses data. Khususnya untuk data yang memiliki basis relasional. Tentu saja bahasa komputer dalam data basis relasional ini berdasarkan standar yang dikeluarkan oleh American National Standard Institute (ANSI). Standardisasi SQL sudah ada sejak tahun 1986 dan memang diinisiasi oleh ANSI.

Hingga saat ini banyak server dalam sebuah database maupun software mampu mengartikan bahasa SQL. Maka dari itulah, SQL menjadi bahan pembicaraan dan sebuah materi yang sangat penting bagi kamu yang bergelut dalam dunia IT maupun hal-hal yang bersinggungan dengan database relasional.

Sejarah kemunculan SQL

Bila kita berbicara mengenai sejarah munculnya dan pengertian SQL dapat mengacu pada artikel dari Jhonny Oracle yang merupakan salah seorang peneliti di IBM. Artikel tersebut membahas mengenai ide pembuatan sebuah basis data relasional yang muncul di tahun 1970, tepatnya pada bulan Juni. Pada artikel tersebut juga dibahas mengenai kemungkinan di masa mendatang untuk menciptakan sebuah bahasa data yang standar. Oleh karena itulah kemudian muncul istilah SEQUEL yang mengacu pada singkatan dari Structured English Query Language.

Selang beberapa waktu, IBM lantas mencanangkan proyek basis data relasional SEQUEL dan karena namanya cukup sulit dieja maka disingkat menjadi SQL. Setelah proses standardisasi selesari tahun 1986, lalu SQL mengalami perbaikan tahun 1989. Tahun 1992 muncul versi SQL92 dan di tahun 1999 dengan berbagai pembaharuan muncullah SQL99. Namun, dalam dunia IT SQL yang sering digunakan mereferensikan pada SQL92. Meski begitu, sampai sekarang pun sebenarnya tidak ada sebuah server berbasis data yang menggunakan maupun mendukung SQL99. Ini lebih tergantung pada dialek masing-masing server yang digunakan.


11. Bagaimana cara membuat program yang memiliki list di dalam nya ,dengan tipe data yg berbeda yaitu,CRUD dan sorting


Jawaban:

Sebuah dokumen database berorientasi , atau toko dokumen , adalah program komputer dan penyimpanan data sistem yang dirancang untuk menyimpan, mengambil dan mengelola informasi berorientasi dokumen, juga dikenal sebagai data semi-terstruktur .

Database berorientasi dokumen adalah salah satu kategori utama database NoSQL , dan popularitas istilah "database berorientasi dokumen" telah berkembang dengan penggunaan istilah NoSQL itu sendiri. Database XML adalah subclass dari database berorientasi dokumen yang dioptimalkan untuk bekerja dengan dokumen XML . Database grafik serupa, tetapi menambahkan lapisan lain, hubungan , yang memungkinkan mereka untuk menautkan dokumen untuk traversal cepat.

Database berorientasi dokumen secara inheren merupakan subkelas dari penyimpanan nilai kunci , konsep database NoSQL lainnya. Perbedaannya terletak pada cara pemrosesan data; di penyimpanan nilai kunci, data dianggap tidak tembus pandang ke database, sedangkan sistem berorientasi dokumen bergantung pada struktur internal dalam dokumen untuk mengekstrak metadata yang digunakan mesin database untuk pengoptimalan lebih lanjut. Meskipun perbedaannya sering diabaikan karena alat dalam sistem, [a] secara konseptual penyimpanan dokumen dirancang untuk menawarkan pengalaman yang lebih kaya dengan teknik pemrograman modern.

Database dokumen [b] sangat kontras dengan database relasional tradisional (RDB). Database relasional umumnya menyimpan data dalam tabel terpisah yang ditentukan oleh programmer, dan satu objek dapat tersebar di beberapa tabel. Database dokumen menyimpan semua informasi untuk objek tertentu dalam satu contoh dalam database, dan setiap objek yang disimpan dapat berbeda satu sama lain. Ini menghilangkan kebutuhan untuk pemetaan relasional objek saat memuat data ke dalam database.

Penjelasan:

Konsep sentral dari database berorientasi dokumen adalah pengertian tentang dokumen . Meskipun setiap implementasi basis data berorientasi dokumen berbeda pada detail definisi ini, secara umum, mereka semua menganggap dokumen mengenkapsulasi dan menyandikan data (atau informasi) dalam beberapa format standar atau penyandian. Pengodean yang digunakan termasuk XML , YAML , JSON , serta bentuk biner seperti BSON .

Dokumen dalam penyimpanan dokumen secara kasar setara dengan konsep pemrograman suatu objek. Mereka tidak diharuskan untuk mematuhi skema standar, juga tidak akan memiliki semua bagian, slot, bagian atau kunci yang sama. Umumnya, program yang menggunakan objek memiliki banyak tipe objek yang berbeda, dan objek tersebut sering kali memiliki banyak bidang opsional. Setiap objek, bahkan yang sekelas, bisa terlihat sangat berbeda. Penyimpanan dokumen serupa karena memungkinkan berbagai jenis dokumen dalam satu penyimpanan, memungkinkan bidang di dalamnya menjadi opsional, dan sering kali memungkinkannya untuk dikodekan menggunakan sistem pengkodean yang berbeda.
